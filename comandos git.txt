comandos git

FIRMA
git config --global user.name "<Tu nombre>"
git config --global user.email "<tu@email.com>"

CHEQUEAR NOMBRE / CORREO
git config user.name / git config user.email

CREAR REPOSITORIO (estando dentro del directorio)
git init

VERIFICAR SI UN DIRECTORIO ES REPOSITORIO
git status (Debería estar asociado a una rama)
ls -d .git (Devuelve la carpeta donde se guarda el historial)

CAMBIAR NOMBRE DE LA RAMA
git branch -M main (-M fuerza el cambio de nombre aunque existan duplicados)

ADVERTENCIA CON CREACIÓN DE REPOSITORIOS EN GITHUB
-No crear repositorio con archivos pre establecidos para evitar conflictos de historial.
-Si se realiza se debe ejecutar un pull para traer los archivos de github

COMANDOS PARA CONECTAR REPOSITORIO LOCAL CON EL DE GITHUB
git remote add origin <link de repositorio en github>
git push -u origin main (vincula la rama local con la remota permanentemente)

main = Rama principal de proyecto por convención

VERIFICAR A QUE REPOSITORIO DE GITHUB ESTÁ CONECTADO EL LOCAL
git remote -v

origin = puntero que conecta los repositorios

IMPORTANTE = Las ramas deben tener el mismo nombre

AÑADIR LAS MODIFICACIONES REALIZADAS EN EL REPO LOCAL
git add .

GIT CONVIERTE LOS ARCHIVOS DE TEXTO EN FORMATO CRLF AL SUBIR Y LF AL DESCARGAR
git config --global core.autocrlf true

CONFIGURAR SSH PARA CONECTAR REPOSITORIOS
ssh-keygen -t ed25519 -C "tuemail@ejemplo.com" (Crear claves ssh pública y privada)

eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519 (Sistema gestiona las claves en 2do plano usando ram)

REGISTRAR CAMBIOS EN UN REPOSITORIO
git commit -m <mensaje de commit>

ENVIAR/TRAER LOS CAMBIOS A/DESDE REPOSITORIO REMOTO
git push origin main/ git pull origin main

ENLAZAR SUBIDAS DE ARCHIVOS A REPOSITORIO REMOTO PERMANENTE
git push -u origin main (luego de esto basta usar git push/pull)

CAMBIAR LA CONEXIÓN DE HTTP A SSH
git remote set-url origin git@github.com:<USUARIO>/<TU-REPOSITORIO>.git

CREAR NUEVA RAMA
git branch <nombre de nueva rama>

REVISAR EN QUE RAMA ME ENCUENTRO
git branch

DESHACER COMMITS
git reset HEAD~1

FORZAR CAMBIOS EN GITHUB
git push --force origin main
